# 赛事详情页生成类型与风格类型调整

## 目标

修正赛事详情页多平台工作台的生成语义，不调整页面结构和现有业务流程。

- 主按钮统一显示“生成”。
- “内容类型”改为“生成类型”。
- “选题方向”改为“风格类型”。
- 选择“选题”时，一次生成 5 个可落地的作品角度，而不是完整脚本或发布稿。
- 其他生成类型继续输出对应的平台内容。

## 交互

用户依次选择热点、生成类型和风格类型，然后点击“生成”。

未点击生成时，内容展示区保持空白。更换热点、生成类型或风格类型后，清空旧结果，避免旧稿与当前配置不一致。

## 生成类型

保留现有类型，但优化名称和模型指令：

- 选题
- 标题
- 短文案
- 视频脚本
- 评论区互动
- 图文卡片

“选题”采用独立结果结构。其余类型保留“可直接发布版、编辑参考版、风险提示版”三层结构。

## 风格类型

风格类型用于控制表达和创作包装，不再承担选题本身：

- 专业复盘
- 客观资讯
- 球迷讨论
- 轻松整活
- 人物故事
- 数据解读
- 稳妥表达

## 选题输出

选择“选题”时，模型必须基于当前选中的热点、比赛事实、事件和数据生成 5 个不同角度。

每个角度仅包含：

1. 角度标题
2. 怎么做
3. 依据或简短说明

角度应优先体现作品形态和创作方法，例如动漫二创、游戏二创、人物关系、时间线、数据卡、名场面拆解或球迷互动。不能只把“专业分析”“赛后复盘”当作角度，也不能直接展开成完整脚本。

没有输入依据时，不生成具体事实断言；涉及伤病、冲突、判罚动机或内部信息时标记需核验。

## 实现范围

- `lib/services/contentService.ts`
  - 更新选项展示文案。
  - 调整无 AI 兜底的选题结果，使其保持 5 个角度和精简结构。
- `lib/ai/platform-draft.ts`
  - 根据生成类型切换提示词和输出解析。
  - 选题类型使用 5 项结构化结果。
- `app/matches/[id]/page.tsx`
  - 更新控件标签和按钮文案。
  - 保持配置变化后清空旧结果。
- `scripts/test-workflow-services.mjs`
  - 验证新标签、5 个选题角度和精简结果结构。

## 验证

1. 先运行工作流服务测试。
2. 运行 `npm.cmd run build`。
3. 检查生成类型、风格类型切换后旧结果会被清空。
4. 确认选题结果正好包含 5 个角度，且不包含完整脚本段落。
